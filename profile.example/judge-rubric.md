# JD Evaluation Rubric (template)

Used by the optional fit-judge: it reads a job description and your rubric, then
returns a verdict (STRONG / DECENT / WEAK / SKIP) with reasons and any hard
blockers. Edit this to describe YOU — the judge is only as good as this file.
You can edit it in-app under Settings → Judge rubric.

## Candidate snapshot

- Years of experience: <e.g. 8+ backend>
- Primary stack: <e.g. TypeScript, Node.js, PostgreSQL>
- Domain: <e.g. fintech, web3, SaaS>
- Location / remote: <e.g. remote-global, or City + visa-sponsorship needs>
- Target roles: <e.g. senior backend IC; EM secondary>

## Auto-skip (any one true → SKIP)

- Primary language is one you don't want (e.g. Go-/Rust-/Java-primary)
- Frontend/mobile as the primary requirement
- Seniority far off (e.g. requires 12+ yrs, or Director/VP scope)
- On-site somewhere you can't go with no visa sponsorship
- Role type you don't want (PM/TPM, pure DevOps, data science, …)

## Score each 0-3 (0 none · 3 strong)

1. Stack match — how many of your core skills the JD actually requires
2. Domain match — your target industries vs the company's
3. Role-type match — IC backend / tech lead / EM, as you prefer
4. Location match — remote/onsite/visa fit
5. Seniority match — years + scope vs yours
6. Compensation signal — disclosed range / funding / equity

## Verdict (total /18)

- STRONG 14-18 · DECENT 10-13 · WEAK 6-9 · SKIP 0-5 (or any auto-skip)

## Output per JD

Verdict + one line why it fits + one line what doesn't + any hard blockers to
verify (e.g. "Go-primary", "onsite, no sponsorship", "requires 10+ yrs").
